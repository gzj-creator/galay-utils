# 07-常见问题

## Q1：为什么 README 不再推荐直接运行 `cmake ..`？

因为当前源码里 `BUILD_MODULE_TESTS` 默认值是 `ON`，而同一个 `CMakeLists.txt` 又要求该选项在 CMake `< 3.28` 时直接报错。为了让 CMake `3.16+` 的基线构建真实可用，文档现在统一使用：

```bash
cmake -S . -B build -DBUILD_MODULE_TESTS=OFF
```

## Q2：这个仓库到底是不是“无外部依赖”？

不能笼统地这么说。

- 只包含部分细粒度头文件时，很多模块确实只依赖标准库 / 平台系统头
- 但 `galay-utils/galay-utils.hpp` 会包含 `RateLimiter.hpp`
- `RateLimiter.hpp` 无条件依赖 `galay-kernel` 与 `concurrentqueue/moodycamel`

因此更准确的说法是：**仓库提供很多 header-only 组件，但 umbrella header / RateLimiter / 模块入口有额外外部头依赖。**

## Q3：`IniParser` 还存在吗？

不存在。当前公开头 `galay-utils/parser/Parser.hpp` 中的真实类型是：

- `ConfigParser`
- `EnvParser`
- `ParserManager`

`.ini` / `.conf` 文件由 `ConfigParser` 负责。

## Q4：真实示例文件和 target 是什么？

- 示例源文件：
  - `examples/include/E1-basic_usage.cpp` → `E1-BasicUsage`
  - `examples/include/E2-system_process.cpp` → `E2-SystemProcess`
  - `examples/include/E3-parser_balancer.cpp` → `E3-ParserBalancer`
- 功能测试入口：`test/test_all.cpp` → `test_all`
- 模块烟雾测试：`test/module_import_smoke.cpp` → `test_import_smoke`
- benchmark 入口：`benchmark/B1-core_benchmark.cpp` → `B1-CoreBench`

其中测试 target 会优先通过 `find_package(galay-kernel CONFIG)` 导入 `galay-kernel::galay-kernel`；在 macOS 默认 `/usr/local` 安装下，`ctest --test-dir build-test` 不需要额外设置 `DYLD_LIBRARY_PATH`。

详情见 [04-示例代码](04-示例代码.md)。

## Q5：什么时候可以用 `import galay.utils;`？

需要同时满足：

- CMake `>= 3.28`
- `Ninja` 或 `Visual Studio`
- 非 `AppleClang`
- 可用的模块扫描工具链

另外，模块接口也会导出 `RateLimiter.hpp`，所以同样需要 `galay-kernel` 与 `concurrentqueue/moodycamel` 头文件。

## Q6：为什么性能页没有吞吐量数字了？

因为仓库虽然现在有 `benchmark/` 与 `B1-CoreBench`，但结果仍依赖机器、编译器和构建参数；所以文档不再把某次运行结果写死成“官方吞吐量表”。当前可信口径是“源码 + 固定命令 + 本地输出”。详情见 [05-性能测试](05-性能测试.md)。

## Q7：哪些组件需要我自己做线程同步？

基于当前实现，至少包括：

- 任何对 `RoundRobinLoadBalancer<T>` 进行并发 `append()` 的场景
- `WeightRoundRobinLoadBalancer<T>` 的共享实例
- `RandomLoadBalancer<T>` 的共享实例
- `WeightedRandomLoadBalancer<T>` 的共享实例

详情见 [06-高级主题](06-高级主题.md)。

## Q8：哪里可以找到最可信的 API 行为说明？

优先顺序如下：

1. `galay-utils/*.hpp`
2. `examples/include/E1-basic_usage.cpp`
3. `test/test_all.cpp`
4. `test/module_import_smoke.cpp`
5. 编号文档页

## Q9：如果我只想用 `StringUtils` / `System`，该怎么避免额外依赖？

不要包含 `galay-utils/galay-utils.hpp`，改为直接包含具体头文件，例如：

- `galay-utils/string/String.hpp`
- `galay-utils/system/System.hpp`

这样可以避免因为 umbrella header 间接引入 `RateLimiter.hpp`。
