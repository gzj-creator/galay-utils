# LRU Cache Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a non-thread-safe, extensible LRU cache with lazy capacity and TTL eviction.

**Architecture:** Implement a header-only `LruCache` template backed by a recency list, lookup map, and optional expiration min-heap with lazy invalidation. Export it through the aggregate include and module facade, then verify behavior through the cache CTest group.

**Tech Stack:** C++23, CMake, CTest, standard library containers.

---

### Task 1: Add failing tests

**Files:**
- Modify: `test/cache/cache_test.cpp`

**Step 1: Write the failing test**

Add `testLruCache()` with assertions for capacity eviction, recency refresh, update behavior, TTL expiration, stale TTL heap nodes, and zero capacity.

**Step 2: Run test to verify it fails**

Run:

```bash
rtk cmake -S . -B cmake-build-test -DBUILD_TESTING=ON -DBUILD_EXAMPLES=OFF
rtk cmake --build cmake-build-test --target cache_test
```

Expected: compile fails because `LruCache` is not defined or its header is missing.

### Task 2: Implement LRU cache header

**Files:**
- Create: `galay-utils/tool/lru_cache.hpp`

**Step 1: Write minimal implementation**

Implement `LruCache<Key, Value, Hash, KeyEqual>` with:

- `std::list<Entry>` for recency
- `std::unordered_map<Key, iterator, Hash, KeyEqual>` for lookup
- `std::priority_queue` min-heap for TTL expiration candidates
- lazy `purgeExpired()` and `enforceCapacity()`
- Doxygen comments that state the class is not thread-safe

**Step 2: Run tests**

Run:

```bash
rtk cmake --build cmake-build-test --target test_all
rtk ctest --test-dir cmake-build-test --output-on-failure
```

Expected: tests pass.
