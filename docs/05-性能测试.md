# 05-性能测试

本页只描述当前仓库里真实存在的 benchmark 资产、固定运行命令和解释边界。

## 1. 当前 benchmark 资产

| 项目 | 当前真实状态 |
|---|---|
| `benchmark/` 目录 | 存在 |
| 顶层开关 | `BUILD_BENCHMARKS` |
| 可直接构建的 target | `B1-CoreBench` |
| benchmark 源码 | `benchmark/B1-core_benchmark.cpp` |
| 默认工作负载 | `--iterations 200000 --repeats 5 --backend-count 8 --config-keys 16` |
| 推荐构建方式 | `Release` |

结论：

- 仓库现在提供可直接运行的 benchmark target
- 旧文档中的历史吞吐量数字仍然不视为当前官方结论
- 当前可信口径是“源码 + 固定命令 + 本地输出结果”，而不是写死在 Markdown 里的旧表格

## 2. 固定运行命令

```bash
cmake -S . -B build-bench \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_TESTS=OFF \
  -DBUILD_BENCHMARKS=ON

cmake --build build-bench --target B1-CoreBench --parallel
./build-bench/benchmark/B1-CoreBench \
  --iterations 200000 \
  --repeats 5 \
  --backend-count 8 \
  --config-keys 16
```

该命令会打印：

- 固定的输入规模
- 当前机器的 `cpuCount`、`hostname`、`executablePath`
- 每个场景的 `best(ns)`、`median(ns)`、`best/op`

## 3. 当前覆盖的 benchmark 场景

| 场景 | 真实来源 | 说明 |
|---|---|---|
| `string.split_join` | `StringUtils::split` / `StringUtils::join` | 固定 CSV 负载的字符串拆分与拼接 |
| `parser.config_parse` | `ConfigParser::parseString` | 固定 `.ini/.conf` 风格文本解析 |
| `balancer.round_robin` | `RoundRobinLoadBalancer<T>::select()` | 固定节点集合上的轮询选择 |
| `balancer.weight_round_robin` | `WeightRoundRobinLoadBalancer<T>::select()` | 固定权重集合上的加权轮询 |
| `system.address_type` | `System::checkAddressType` | IPv4 / IPv6 / 域名判定 |

## 4. 这套 benchmark 能证明什么

- 能提供同机、同命令、同输入规模下的回归基线
- 能作为“修改前后是否明显退化”的快速信号
- 能把性能说明从“历史文案”提升为“源码可复现”

## 5. 这套 benchmark 还不能证明什么

- 不能直接代表跨机器、跨编译器、跨操作系统的统一吞吐量承诺
- 还没有覆盖 `RateLimiter`、`CircuitBreaker`、`ThreadPool` 等带外部依赖或调度语义更复杂的模块
- 还没有把 benchmark 输出自动写入 CI 产物

## 6. 当前已知性能相关限制

- `RandomLoadBalancer` / `WeightedRandomLoadBalancer` 使用共享 RNG；`RoundRobinLoadBalancer::append()` 也没有内部同步，共享实例的多线程修改仍需外部同步
- `BlockingObjectPool::acquire()`、`ThreadPool::waitAll()`、`TaskWaiter::wait()` 是阻塞接口，不适合直接放进协程调度线程
- `BUILD_MODULE_TESTS` 默认开启会影响基线构建成功率，应与 benchmark 链路分开配置

## 7. 历史说明

旧版文档里出现过大量性能表，但那些数值没有与当前仓库中的源码、target 和固定命令一一对应。它们现在仍只被视为历史文案，不再作为当前性能声明。
