# 05-性能测试

本页只描述当前仓库里真实存在的 benchmark 资产与边界。当前结论很简单：仓库没有受版本控制的 benchmark 目录或 benchmark target。

| 项目 | 当前真实状态 |
|---|---|

结论：

- 旧文档中的 benchmark、吞吐量表和运行命令都不再代表当前仓库真相
- 当前可信口径是“没有受版本控制的 benchmark 资产”，因此不对外发布性能承诺

## 2. 为什么保留这一页

保留这一页是为了明确告诉读者：

- 当前仓库没有受版本控制的 benchmark 目录
- README 与 numbered docs 中任何旧 benchmark 说法都应视为历史文案
- 如果未来重新引入 benchmark，应以真实源码目录、真实 target、真实固定命令为准

## 3. 当前已知性能相关限制

- `RandomLoadBalancer` / `WeightedRandomLoadBalancer` 使用共享 RNG；`RoundRobinLoadBalancer::append()` 也没有内部同步，共享实例的多线程修改仍需外部同步
- `BlockingObjectPool::acquire()`、`ThreadPool::waitAll()`、`TaskWaiter::wait()` 是阻塞接口，不适合直接放进协程调度线程
- `BUILD_MODULE_TESTS` 默认关闭；只有在明确验证模块导入时才建议开启

## 4. 历史说明

旧版文档里出现过大量性能表，但那些数值没有与当前仓库中的源码、target 和固定命令一一对应。它们现在仍只被视为历史文案，不再作为当前性能声明。
