# 01-架构设计

本页描述当前仓库实际导出的入口、目录职责与依赖边界。

## 1. 导出面

| 入口 | 真实文件 / target | 说明 |
|---|---|---|
| 细粒度头文件 | `galay-utils/<module>/*.hpp` | 最稳妥的接入面，依赖边界最清晰 |
| umbrella header | `galay-utils/galay_utils.hpp` | 一次性聚合常用模块，不再默认导出 `limiter.hpp` |
| CMake 接口库 | target `galay-utils` | `INTERFACE` 库，导出 include 目录与 Unix 链接要求 |
| C++23 模块 | `galay-utils/module/galay_utils.cppm` / target `galay-utils-modules` | 仅在工具链满足条件时生成 |

## 2. 目录职责

| 路径 | 角色 |
|---|---|
| `galay-utils/` | 公开头文件与模块接口 |
| `examples/` | 真实可构建示例；当前提供 `E1-BasicUsage`，满足模块条件时额外提供 `E1-BasicUsageImport` |
| `test/` | 真实断言与模块烟雾测试 |
| `docs/` | 人类可读文档；当与源码冲突时优先级最低 |
| `cmake/option.cmake` | 平台 IO 后端与构建选项 |

## 3. 依赖边界

### 3.1 CMake target 级别

- `galay-utils` 是 `INTERFACE` 库
- Unix 下自动附带 `pthread`
- Linux 下额外附带 `dl`

### 3.2 头文件级别

- 大多数模块只依赖标准库或平台系统头
- `galay-utils/ratelimiter/limiter.hpp` 仅依赖标准库，提供同步非阻塞 `tryAcquire` 路径
- `galay-utils/galay_utils.hpp` 不再默认包含 `limiter.hpp`
- `galay-utils/module/galay_utils.cppm` 也不再默认导出 `limiter.hpp`

结论：

- “限流器不再把 `galay-kernel` 依赖带入 `galay-utils`”
- “只想用轻量模块时，细粒度头文件仍然是最稳妥的边界”

## 4. 构建开关

| 选项 | 默认值 | 真实行为 |
|---|---|---|
| `BUILD_EXAMPLES` | `ON` | 构建 `examples/CMakeLists.txt` 中的 target |
| `BUILD_TESTING` | `OFF` | 开启后构建 `test_all`，并在满足条件时构建 `test_import_smoke` |
| `BUILD_TESTS` | 兼容别名 | 传入时会映射到 `BUILD_TESTING` |
| `BUILD_MODULE_TESTS` | `OFF` | 显式开启后才会构建模块 target、模块示例和模块烟雾测试 |
| `DISABLE_IOURING` | `ON` | Linux 默认走 `epoll`；显式关闭后才尝试 `io_uring` |

## 5. 验证资产

| 资产 | 真实文件 | 作用 |
|---|---|---|
| 示例 | `examples/include/e1_basic.cpp`、`examples/import/e1_basic.cpp` | 覆盖 include 与 import 两条基础接入路径 |
| 功能测试 | `test/test_all.cpp` | 大多数模块的行为校验基线 |
| 模块烟雾测试 | `test/import_smoke.cpp` | `import galay.utils;` 的最小验证 |

## 6. 平台与特性边界

- `BackTrace` 只在 Apple/Linux 上提供实际栈采集逻辑
- `Process::daemonize()` 仅对 POSIX 语义有意义
- `SignalHandler` 修改的是进程级信号处理器
- 模块构建不支持 `AppleClang`

## 7. 当前权衡

- 仓库同时提供“细粒度头文件”和“全量 umbrella header”，但后者的依赖边界更宽
- 当前没有受版本控制的 benchmark 目录，因此本仓库不对外给出“真实 benchmark target”承诺
- `examples/` 当前覆盖 include/import 两条基础接入路径；更深的边界与异常路径仍主要来自 `test/test_all.cpp`
