# 03-使用指南

本页做“怎么选模块”的决策说明，不承担示例代码真相来源。真实可运行示例请看 [04-示例代码](04-示例代码.md)。

## 1. 先选头文件策略

| 场景 | 推荐做法 | 原因 |
|---|---|---|
| 只需要少量工具类 | 直接包含具体头文件 | 依赖边界最小 |
| 想快速接入常用模块 | 可以使用 `galay-utils/galay_utils.hpp` | 接入简单，且不再默认拉入限流器依赖 |
| 需要限流器家族 | 直接包含 `galay-utils/tool/rate_limiter.hpp` | 无锁同步非阻塞接口，无 `galay-kernel` 依赖 |
| 已具备模块工具链 | 使用 `import galay.utils;` | 导入体验更好，但约束最多 |

## 2. 常见需求到模块映射

| 需求 | 模块 |
|---|---|
| 字符串拆分、大小写、十六进制 | `StringUtils` |
| 本地随机数、随机字符串、UUID | `RandomGenerator` |

| 时间戳、秒表、截止时间、退避 | `Time`、`StopWatch`、`Deadline`、`Backoff` |
| 文件、目录、环境变量、主机信息 | `System` |
| 容量缓存与 TTL 缓存 | `LruCache` |
| 仅移动字节容器 / 原始字节元数据 | `Bytes`、`ByteMetaData` |
| 流式字节队列 | `ByteQueueView` |
| 固定容量环形缓冲 | `RingBuffer` |
| 线程池与批量等待 | `ThreadPool`、`TaskWaiter` |
| 轻量对象复用 | `ObjectPool<T>` |
| 需要阻塞等待资源 | `BlockingObjectPool<T>` |
| 简单并发限制 | `CountingSemaphore` |
| 允许突发流量 | `TokenBucketLimiter` |
| 固定窗口内限次 | `SlidingWindowLimiter` |
| 平滑流出控制 | `LeakyBucketLimiter` |
| 熔断与降级 | `CircuitBreaker` |
| 轮询 / 加权 / 随机负载均衡 | `Balancer` 系列 |

| 命令行参数 | `App` / `Cmd` / `Arg` |
| `.ini` / `.conf` / `.env` / `.toml` 配置 | `ConfigParser`、`IniParser`、`EnvParser`、`TomlParser`、`ParserManager` |

## 3. RateLimiter 怎么选

| 类型 | 更适合什么 | 备注 |
|---|---|---|
| `CountingSemaphore` | 控制并发数量 | API 最直接 |
| `TokenBucketLimiter` | 接口限流且允许突发 | 适合通用配额 |
| `SlidingWindowLimiter` | 更接近固定时间窗配额 | 无参数化 `acquire` |
| `LeakyBucketLimiter` | 流量整形 | 更强调平滑流出 |

注意：

- 这四个类型都定义在 `galay-utils/tool/rate_limiter.hpp`
- 仅提供无锁同步非阻塞 `tryAcquire` 路径，未通过限流时直接返回 `false`
- 不再提供协程 `acquire()` awaitable；协程等待、重试、超时策略应放在上层项目自行适配

## 4. LoadBalancer 怎么选

| 类型 | 推荐场景 | 并发提示 |
|---|---|---|
| `RoundRobinLoadBalancer<T>` | 节点能力接近、节点集大多固定 | 仅 `select()` 在节点集合不再变更时适合并发共享；并发 `append()` 需外部同步 |
| `WeightRoundRobinLoadBalancer<T>` | 需要平滑按权重分配 | 内部状态可变，外部自行同步 |
| `RandomLoadBalancer<T>` | 随机分配即可 | 内部 RNG 无锁，外部自行同步 |
| `WeightedRandomLoadBalancer<T>` | 按权重随机 | 内部 RNG 无锁，外部自行同步 |

以上并发说明是基于当前 `balancer.inl` 实现得出的结论。

补充：

- 四个 `select()` 都返回 `std::optional<T>` 按值，因此这里不是零拷贝 API

## 5. BloomFilter 怎么用

| 判断结果 | 含义 | 后续动作 |
|---|---|---|
| `possiblyContains(x) == false` | 一定不存在 | 可以直接过滤掉 |
| `possiblyContains(x) == true` | 可能存在，可能是假阳性 | 需要精确结果时回源确认 |

注意：

- `BloomFilter` 不提供 `contains()`，避免误导成精确集合。
- 不支持删除；需要删除能力时应选择 Counting Bloom Filter 或精确集合。
- false positive rate 取决于容量、插入量和 hash 分布；插入超出预期会明显提高误判率。

## 6. Pool / Thread 怎么选

| 需求 | 推荐模块 | 备注 |
|---|---|---|
| 异步执行很多小任务 | `ThreadPool` | `waitAll()` 会阻塞线程 |
| 等待一批异步工作 | `TaskWaiter` | `wait()` / `waitFor()` 会阻塞线程 |
| 非阻塞对象复用 | `ObjectPool<T>` | 池空时按需创建新对象 |
| 池空时必须等待 | `BlockingObjectPool<T>` | `acquire()` / `tryAcquireFor()` 会阻塞线程 |

## 7. Parser / App 怎么选

| 需求 | 推荐模块 | 备注 |
|---|---|---|
| 解析 `.conf` | `ConfigParser` | 支持 section 与 `section.key` |
| 解析 `.ini` | `IniParser` | 行为与 `ConfigParser` 对齐，保留独立类型 |
| 解析 `.env` | `EnvParser` | 面向 `KEY=VALUE` |
| 解析 `.toml` | `TomlParser` | 支持基础 section、dotted key 和数组 |
| 根据扩展名选择解析器 | `ParserManager` | 默认识别 `.conf` / `.ini` / `.env` / `.toml` |
| 构建带子命令 CLI | `App` / `Cmd` / `Arg` | 支持长短参数、默认值、flag、子命令 |
