# 00-快速开始

本页只保留和当前仓库源码一致的接入路径、target 名称与前置条件。

## 1. 先选接入方式

| 目标 | 推荐入口 | 是否需要额外头依赖 |
|---|---|---|
| 只用单个轻量模块 | 直接包含具体头文件，例如 `galay-utils/string/String.hpp` | 通常不需要 |
| 想一次性包含所有模块 | `galay-utils/galay-utils.hpp` | 需要 `galay-kernel` 与 `concurrentqueue/moodycamel` |
| 想用 C++23 模块 | `import galay.utils;` | 需要模块工具链，且同样需要 `galay-kernel` 与 `concurrentqueue/moodycamel` |

## 2. 基线构建

前置条件：

- C++23 编译器
- CMake `>= 3.16`
- 若 CMake `< 3.28`，请显式关闭 `BUILD_MODULE_TESTS`
- 环境变量：无

```bash
git clone https://github.com/gzj-creator/galay-utils.git
cd galay-utils

cmake -S . -B build \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_TESTS=OFF

cmake --build build --parallel
```

说明：

- `CMakeLists.txt` 的默认 `BUILD_MODULE_TESTS=ON`
- 但源码同时要求 `BUILD_MODULE_TESTS` 在 CMake `< 3.28` 时直接报错
- 因此“基线构建”应当把该选项显式关闭

## 3. 构建真实示例 `E1-BasicUsage`

真实资产：

- 源文件：`examples/include/E1-basic_usage.cpp`
- target：`E1-BasicUsage`
- 环境变量：无

前置条件：

- C++23 编译器
- CMake `>= 3.16`
- `galay-kernel` 头文件可见
- `concurrentqueue/moodycamel` 头文件可见

```bash
cmake -S . -B build-examples \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTS=OFF

cmake --build build-examples --target E1-BasicUsage --parallel
./build-examples/examples/E1-BasicUsage
```

## 4. 构建真实测试 `test_all`

真实资产：

- 源文件：`test/test_all.cpp`
- target：`test_all`
- 环境变量：无

前置条件：

- C++23 编译器
- CMake `>= 3.16`
- `galay-kernel` 头文件与库可用；若不在默认前缀，请通过 `-DCMAKE_PREFIX_PATH=/path/to/prefix` 让 CMake 发现 `galay-kernel-config.cmake`
- `concurrentqueue/moodycamel` 头文件可见

```bash
cmake -S . -B build-test \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_TESTS=ON

cmake --build build-test --target test_all --parallel
ctest --test-dir build-test --output-on-failure
```

说明：

- `test/CMakeLists.txt` 会优先导入 `galay-kernel::galay-kernel`
- macOS 上会把该共享库目录写入 `test_all` / `test_import_smoke` 的 `BUILD_RPATH`
- 默认 `/usr/local` 安装下，不需要额外设置 `DYLD_LIBRARY_PATH`

## 5. 构建模块烟雾测试 `test_import_smoke`

真实资产：

- 源文件：`test/module_import_smoke.cpp`
- target：`test_import_smoke`
- 导入面：`import galay.utils;`
- 环境变量：无

前置条件：

- CMake `>= 3.28`
- `Ninja` 或 `Visual Studio`
- 非 `AppleClang`
- C++23 模块扫描可用
- `galay-kernel` 头文件与库可用
- `concurrentqueue/moodycamel` 头文件可见

```bash
cmake -S . -B build-mod -G Ninja \
  -DCMAKE_CXX_COMPILER=clang++ \
  -DBUILD_TESTS=ON \
  -DBUILD_MODULE_TESTS=ON \
  -DBUILD_EXAMPLES=OFF

cmake --build build-mod --target test_import_smoke --parallel
ctest --test-dir build-mod -R ModuleImportSmoke --output-on-failure
```

## 6. 在你的项目中集成

### 方式 A：`add_subdirectory`

```cmake
add_subdirectory(third_party/galay-utils)
target_link_libraries(your_target PRIVATE galay-utils)
```

### 方式 B：`find_package`

```cmake
find_package(galay-utils REQUIRED)
target_link_libraries(your_target PRIVATE galay::galay-utils)
```

### 方式 C：只拷贝头文件

如果你不想引入 umbrella header，可只复制 `galay-utils/` 目录并在业务代码中包含需要的具体头文件。

## 7. 接入建议

- 想最小化依赖面：优先直接包含具体头文件
- 想复用仓库现成示例：优先看 [04-示例代码](04-示例代码.md) 中的三个示例 target
- 想验证 API 是否可运行：使用 `test_all`
- 想验证模块导入：使用 `test_import_smoke`
