# 00-快速开始

本页只保留和当前仓库源码一致的接入路径、target 名称与前置条件。

## 1. 先选接入方式

| 目标 | 推荐入口 | 是否需要额外头依赖 |
|---|---|---|
| 只用单个轻量模块 | 直接包含具体头文件，例如 `galay-utils/core/string.hpp` | 通常不需要 |
| 想一次性包含常用模块 | `galay-utils/galay_utils.hpp` | 需要能找到 `concurrentqueue` 头文件 |
| 想用缓存或缓冲工具 | `galay-utils/cache/lru_cache.hpp`、`galay-utils/cache/bytes.hpp`、`galay-utils/cache/byte_queue_view.hpp`、`galay-utils/cache/ring_buffer.hpp` | 不需要额外第三方头依赖 |
| 想用限流器 | `galay-utils/tool/rate_limiter.hpp` | 不需要额外第三方头依赖 |
| 想用线程池 | `galay-utils/tool/thread.hpp` | 需要能找到 `concurrentqueue` 头文件 |
| 想用 C++23 模块 | `import galay.utils;` | 需要模块工具链 |

## 2. 基线构建

前置条件：

- C++23 编译器
- CMake `>= 3.16`
- 第三方头文件：需要能找到 `<concurrentqueue/moodycamel/blockingconcurrentqueue.h>`
- 模块验证默认关闭；需要时再显式开启 `BUILD_MODULE_TESTS`
- 环境变量：无

```bash
git clone https://github.com/gzj-creator/galay-utils.git
cd galay-utils

cmake -S . -B build \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_TESTING=OFF

cmake --build build --parallel
```

说明：

- `BUILD_MODULE_TESTS` 默认 `OFF`
- 基线构建不需要 `galay-kernel`
- 如果 `concurrentqueue` 不在常见系统路径，传入 `-DGALAY_UTILS_CONCURRENTQUEUE_INCLUDE_DIR=/path/to/include`

## 3. 构建真实示例 `E1-BasicUsage`

真实资产：

- 源文件：`examples/include/e1_basic.cpp`
- target：`E1-BasicUsage`
- 环境变量：无

前置条件：

- C++23 编译器
- CMake `>= 3.16`
- 需要能找到 `concurrentqueue` 头文件，因为 umbrella header 包含 `ThreadPool`

```bash
cmake -S . -B build-examples \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTING=OFF

cmake --build build-examples --target E1-BasicUsage --parallel
./build-examples/examples/E1-BasicUsage
```

## 4. 构建真实测试

真实资产：

- 源文件：`test/<area>/*_test.cpp`
- target：`algorithm_test`、`app_test`、`buffer_test`、`cache_test`、`concurrency_test`、`core_test`、`data_test`、`platform_test`、`resilience_test`、`routing_test`
- 环境变量：无

前置条件：

- C++23 编译器
- CMake `>= 3.16`
- `resilience_test` 会额外直接包含 `galay-utils/tool/rate_limiter.hpp`
- `concurrency_test` 需要能找到 `concurrentqueue` 头文件

```bash
cmake -S . -B build-test \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_TESTING=ON

cmake --build build-test --parallel
ctest --test-dir build-test --output-on-failure
```

说明：

- `test/CMakeLists.txt` 只链接 `galay-utils`
- 不需要额外设置 `DYLD_LIBRARY_PATH`

## 5. 构建模块烟雾测试 `test_import_smoke`

真实资产：

- 源文件：`test/import_smoke.cpp`
- target：`test_import_smoke`
- 导入面：`import galay.utils;`
- 环境变量：无

前置条件：

- CMake `>= 3.28`
- `Ninja` 或 `Visual Studio`
- 非 `AppleClang`
- C++23 模块扫描可用
- 限流器仍通过细粒度头显式包含；无额外第三方头依赖

```bash
cmake -S . -B build-mod -G Ninja \
  -DCMAKE_CXX_COMPILER=clang++ \
  -DBUILD_TESTING=ON \
  -DBUILD_MODULE_TESTS=ON \
  -DBUILD_EXAMPLES=OFF

cmake --build build-mod --target test_import_smoke --parallel
ctest --test-dir build-mod -R ModuleImportSmoke --output-on-failure
```

## 6. 在你的项目中集成

### 方式 A：`add_subdirectory`

```cmake
add_subdirectory(third_party/galay-utils)
target_link_libraries(your_target PRIVATE galay-utils)
```

### 方式 B：`find_package`

```cmake
find_package(galay-utils REQUIRED)
target_link_libraries(your_target PRIVATE galay::galay-utils)
```

### 方式 C：只拷贝头文件

如果你不想引入 umbrella header，可只复制 `galay-utils/` 目录并在业务代码中包含需要的具体头文件。

## 7. 接入建议

- 想最小化依赖面：优先直接包含具体头文件
- 想复用仓库现成示例：优先看 [04-示例代码](04-示例代码.md) 中的 include/import 示例
- 想验证 API 是否可运行：使用按模块拆分的 `*_test`
- 想验证模块导入：使用 `test_import_smoke`
