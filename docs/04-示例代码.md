# 04-示例代码

本页只收录能定位到真实文件与 target 的示例来源。编号文档负责 canonical 说明；这里负责告诉你“哪份源码可以直接构建运行”。

## 2. `examples/` 中的真实示例

统一构建命令：

```bash
cmake -S . -B build-examples \
  -DBUILD_MODULE_TESTS=OFF \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTING=OFF
```

### 2.1 `E1-BasicUsage`

- 源文件：`examples/include/E1-basic_usage.cpp`
- 覆盖模块：`StringUtils`、`System`
- 适合场景：验证 umbrella header 接入是否打通
- 额外前置：无额外第三方头依赖

```bash
cmake --build build-examples --target E1-BasicUsage --parallel
./build-examples/examples/E1-BasicUsage
```

### 2.2 `E1-BasicUsageImport`

- 源文件：`examples/import/E1-basic_usage.cpp`
- 覆盖模块：`StringUtils`、`System`
- 适合场景：验证 `import galay.utils;` 的最小消费路径
- 额外前置：CMake `>= 3.28`、支持模块扫描的生成器/编译器，并显式开启 `BUILD_MODULE_TESTS`

```bash
cmake --build build-examples --target E1-BasicUsageImport --parallel
./build-examples/examples/E1-BasicUsageImport
```

## 3. 测试驱动的示例来源

`test/test_all.cpp` 仍然是覆盖面最广的行为来源，但它不再是“除了唯一示例之外的唯一代码样本”。

运行说明：

- `test/CMakeLists.txt` 会优先使用 `find_package(galay-kernel CONFIG)` 导入 `galay-kernel::galay-kernel`
- macOS 会把 `galay-kernel` 的库目录写入测试 target 的 `BUILD_RPATH`
- 默认 `/usr/local` 安装下，`ctest --test-dir build-test` 不需要额外 loader 环境变量

|---|---|---|---|

| `ObjectPool` / `BlockingObjectPool` | `test/test_all.cpp` | `test_all` | 覆盖获取、释放、超时 |

## 4. 模块导入示例

- 源文件：`test/module_import_smoke.cpp`
- target：`test_import_smoke`
- 环境变量：无

前置条件：

- CMake `>= 3.28`
- `Ninja` 或 `Visual Studio`
- 非 `AppleClang`
- 无额外第三方头依赖

当前烟雾测试覆盖：

- `import galay.utils;`
- `StringUtils::split` / `join`
- `System::cpuCount`

## 5. 本页不再做的事情

- 不把没有对应源文件的长片段标成“官方示例”
- 不把 `IniParser` 当成真实类型名；公开头中的类型是 `ConfigParser`
- 不用不存在的 target 或目录名描述运行方式
